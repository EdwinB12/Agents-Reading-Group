# Ideas & Suggestions

This folder is for capturing informal ideas for future reading group topics — papers to read, tools to demo, exercises to try, open questions to discuss. Ideas do not need to be fully formed; a rough title and a sentence or two is enough to get started.

## What belongs here

- Papers or preprints worth discussing
- Tools, frameworks, or demos to explore hands-on
- Exercise or workshop concepts
- Open questions you'd like the group to tackle
- Links to talks, blog posts, or threads that sparked an idea

## Naming convention

Use the same pattern as `sessions/`:

```
YYYY-MM-DD-short-slug.md
```

Use today's date when you create the file. Example: `2026-04-10-multi-agent-evals.md`

## Template

Copy this into your new idea file:

```markdown
# [Title of idea]

**Proposed by:** [your name / GitHub handle]
**Date added:** YYYY-MM-DD
**Suggested format:** paper / demo / exercise / discussion (pick one or combine)

## Why interesting

One or two sentences on why this would be valuable for the group.

## Links

- [Primary resource](https://...)
- [Supporting material](https://...)

## Notes

Any extra context, prerequisites, or open questions. Leave blank if not applicable.
```

## From idea to session

When an idea is ready to become a full session plan, copy the session template from `CONTRIBUTING.md`, create a file under `sessions/`, and open a PR. You can delete the idea file in the same PR or leave it for reference.
